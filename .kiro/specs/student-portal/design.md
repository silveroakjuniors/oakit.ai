# Design Document ‚Äî Student Portal

## Overview

The Student Portal adds a `student` role to the Oakit platform. It is built on top of the existing infrastructure: the Node.js/Express API gateway, the Python AI service, the Next.js frontend, and the PostgreSQL database. The design reuses existing patterns (JWT auth, role guards, AI query pipeline) and adds new tables, routes, and UI pages with minimal disruption to existing code.

## Database Schema

### `student_portal_config`
Controls which classes have the student portal enabled.

```sql
CREATE TABLE student_portal_config (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  school_id   UUID NOT NULL REFERENCES schools(id) ON DELETE CASCADE,
  class_id    UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  enabled     BOOLEAN NOT NULL DEFAULT false,
  enabled_at  TIMESTAMPTZ,
  updated_by  UUID REFERENCES users(id),
  updated_at  TIMESTAMPTZ DEFAULT now(),
  UNIQUE (school_id, class_id)
);
```

### `student_accounts`
Login credentials for students. Separate from the `users` table to avoid role confusion.

```sql
CREATE TABLE student_accounts (
  id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  school_id            UUID NOT NULL REFERENCES schools(id) ON DELETE CASCADE,
  student_id           UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE UNIQUE,
  username             TEXT NOT NULL,
  password_hash        TEXT NOT NULL,
  force_password_reset BOOLEAN NOT NULL DEFAULT true,
  is_active            BOOLEAN NOT NULL DEFAULT true,
  created_by           UUID REFERENCES users(id),
  created_at           TIMESTAMPTZ DEFAULT now(),
  updated_at           TIMESTAMPTZ DEFAULT now(),
  UNIQUE (school_id, username)
);
CREATE INDEX ON student_accounts(school_id, username);
```

### `quizzes`
A quiz is either self-initiated by a student or assigned by a teacher.

```sql
CREATE TABLE quizzes (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  school_id       UUID NOT NULL REFERENCES schools(id) ON DELETE CASCADE,
  section_id      UUID NOT NULL REFERENCES sections(id) ON DELETE CASCADE,
  created_by_role TEXT NOT NULL CHECK (created_by_role IN ('student', 'teacher')),
  created_by_id   UUID NOT NULL,
  subject         TEXT NOT NULL,
  date_from       DATE NOT NULL,
  date_to         DATE NOT NULL,
  topic_ids       UUID[] NOT NULL DEFAULT '{}',
  question_types  TEXT[] NOT NULL DEFAULT '{}',
  time_limit_mins INT,
  is_assigned     BOOLEAN NOT NULL DEFAULT false,
  due_date        DATE,
  status          TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft','active','closed')),
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

### `quiz_questions`
Questions generated by AI for a quiz.

```sql
CREATE TABLE quiz_questions (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  quiz_id      UUID NOT NULL REFERENCES quizzes(id) ON DELETE CASCADE,
  chunk_id     UUID REFERENCES curriculum_chunks(id),
  subject      TEXT,
  question     TEXT NOT NULL,
  q_type       TEXT NOT NULL CHECK (q_type IN ('fill_blank','descriptive','1_mark','2_mark')),
  marks        INT NOT NULL DEFAULT 1,
  answer_key   TEXT NOT NULL,
  explanation  TEXT,
  position     INT NOT NULL DEFAULT 0
);
```

### `quiz_attempts`
One row per student per quiz attempt.

```sql
CREATE TABLE quiz_attempts (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  quiz_id         UUID NOT NULL REFERENCES quizzes(id) ON DELETE CASCADE,
  student_id      UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  school_id       UUID NOT NULL REFERENCES schools(id) ON DELETE CASCADE,
  started_at      TIMESTAMPTZ DEFAULT now(),
  submitted_at    TIMESTAMPTZ,
  total_marks     INT NOT NULL DEFAULT 0,
  scored_marks    INT NOT NULL DEFAULT 0,
  status          TEXT NOT NULL DEFAULT 'in_progress' CHECK (status IN ('in_progress','submitted','timed_out')),
  UNIQUE (quiz_id, student_id)
);
```

### `quiz_answers`
One row per question per attempt.

```sql
CREATE TABLE quiz_answers (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  attempt_id      UUID NOT NULL REFERENCES quiz_attempts(id) ON DELETE CASCADE,
  question_id     UUID NOT NULL REFERENCES quiz_questions(id) ON DELETE CASCADE,
  student_answer  TEXT,
  is_correct      BOOLEAN,
  marks_awarded   INT NOT NULL DEFAULT 0,
  ai_feedback     TEXT
);
```

## API Routes

### Auth

**`POST /api/v1/auth/student-login`**
- Body: `{ school_code, username, password }`
- Looks up `student_accounts` by `(school_id, username)`
- Checks `student_portal_config.enabled` for the student's class
- Returns JWT with `role: "student"`, `student_id`, `section_id`, `school_id`
- Sets `force_password_reset` flag in token if account has default password

### Admin ‚Äî Portal Config

**`GET /api/v1/admin/student-portal/config`**
Returns enabled/disabled status for all classes in the school.

**`PUT /api/v1/admin/student-portal/config/:classId`**
Body: `{ enabled: boolean }`
Upserts `student_portal_config` for the class.

### Admin / Teacher ‚Äî Credential Management

**`POST /api/v1/teacher/students/credentials/generate`**
Body: `{ student_id }` or `{ section_id }` (bulk)
- Generates username: `{firstname}-{classname}-{section}` lowercased, hyphenated
- Handles collisions by appending `-2`, `-3`, etc.
- Hashes default password `123456` with bcrypt
- Upserts into `student_accounts` (resets password if account exists)
- Returns `{ username, password: "123456" }` (plaintext shown once)

**`POST /api/v1/teacher/students/credentials/reset/:studentId`**
Resets password to `123456`, sets `force_password_reset = true`.

**`GET /api/v1/teacher/students/credentials/:sectionId`**
Returns list of students with their username and account status.

### Student ‚Äî Auth & Profile

**`POST /api/v1/auth/change-password`** (existing, extended to handle `role: "student"`)
Looks up `student_accounts` instead of `users` when role is `student`.

### Student ‚Äî Daily Feed

**`GET /api/v1/student/feed?date=YYYY-MM-DD`**
- Validates date is not more than 5 school days ahead
- Returns `daily_completions` + `curriculum_chunks` for the student's section on that date
- Returns `teacher_homework` for the date
- Returns `teacher_notes` for the date

**`GET /api/v1/student/feed/summary?from=YYYY-MM-DD&to=YYYY-MM-DD`**
Returns all covered topics grouped by subject for the date range.

**`GET /api/v1/student/homework/history`**
Returns last 30 days of `teacher_homework` + `homework_submissions` for the student.

### Student ‚Äî AI Doubts

**`POST /api/v1/ai/student-query`**
- Body: `{ text, student_id }`
- Builds context: all `covered_chunk_ids` for the student's section up to today
- Forwards to AI service `/internal/student-query` with covered topic context
- AI checks relevance before answering

### Student ‚Äî Quiz

**`GET /api/v1/student/quiz/topics?subject=&from=&to=`**
Returns covered topics for the subject in the date range (for topic confirmation step).

**`POST /api/v1/student/quiz/generate`**
Body: `{ subject, topic_ids, question_types, date_from, date_to }`
- Creates `quizzes` record with `is_assigned: false`
- Calls AI service `/internal/generate-quiz`
- Stores questions in `quiz_questions`
- Returns quiz with questions (no answer keys exposed)

**`POST /api/v1/student/quiz/:quizId/start`**
Creates `quiz_attempts` record, returns questions without answer keys.

**`POST /api/v1/student/quiz/:quizId/submit`**
Body: `{ answers: [{ question_id, answer }] }`
- Calls AI service `/internal/evaluate-quiz`
- Stores `quiz_answers` with marks and feedback
- Updates `quiz_attempts` with total/scored marks and `submitted_at`
- Returns full result with correct answers and explanations

**`GET /api/v1/student/quiz/results`**
Returns all `quiz_attempts` for the student with summary stats.

### Teacher ‚Äî Quiz Assignment

**`POST /api/v1/teacher/quiz/assign`**
Body: `{ section_id, subject, topic_ids, question_types, time_limit_mins, due_date, date_from, date_to }`
- Creates `quizzes` record with `is_assigned: true`
- Generates questions via AI
- Returns quiz for teacher review before activation

**`POST /api/v1/teacher/quiz/:quizId/activate`**
Sets `status: "active"` ‚Äî students can now see and take the test.

**`GET /api/v1/teacher/quiz/analytics/:sectionId`**
Returns per-student quiz results, averages, weak areas for the section.

### Parent ‚Äî Analytics

**`GET /api/v1/parent/student-analytics/:studentId`**
- Checks `student_portal_config.enabled` for the student's class
- Returns quiz history, marks, weak areas for the student

## Username Generation Algorithm

```
firstname = student.name.split(' ')[0].toLowerCase()
classname = class.name.toLowerCase().replace(/\s+/g, '')   // "lkg", "ukg", "prep1"
section   = section.label.toLowerCase()                    // "a", "b"
base      = `${firstname}-${classname}-${section}`         // "aarav-lkg-a"

// Collision resolution
candidate = base
suffix = 2
while (exists(school_id, candidate)):
    candidate = `${base}-${suffix}`
    suffix++
username = candidate
```

## Correctness Properties (Property-Based Testing)

1. **Username uniqueness**: For any two students in the same school, generated usernames must differ.
2. **Portal gate**: A student whose class has `enabled = false` must never receive a valid JWT from student-login.
3. **AI scope**: The student-query endpoint must never return a response that references a `curriculum_chunk` not in the student's `covered_chunk_ids`.
4. **Quiz isolation**: During an assigned test attempt, the student-query endpoint must return a blocked response regardless of the question asked.
5. **Score integrity**: `quiz_attempts.scored_marks` must always equal the sum of `quiz_answers.marks_awarded` for that attempt.
6. **Force reset gate**: A student with `force_password_reset = true` must receive a 403 on all routes except `POST /api/v1/auth/change-password`.
